## 一、我建议你的自动化体系：3 个引擎

### 1）主引擎：micro225 日内方向策略

这是最适合先做出来的版本。

**核心思路**
不用做极短秒级高频，而做**开盘/夜盘开盘后的短趋势跟随 + 盘中均值回归**。

#### A. 开盘突破子策略

适用时段：

* 日盘：8:45–10:30
* 夜盘：17:00–20:00

逻辑：

1. 开盘后先观察前 3~5 分钟，形成 opening range。
2. 如果价格向上突破 opening range 高点，同时满足：

   * 1 分钟均线向上
   * 价格在 VWAP 上方
   * NT 倍率同步走强，说明不是只有 TOPIX 在动，而是日经侧更强
     就做多。
3. 反过来，向下突破则做空。
4. 止损不要太小，建议从 **6~10 tick** 起步；止盈建议 **10~20 tick** 或移动止盈。
5. **不要做 1 tick 止盈**，因为 micro225 的成本结构太挤。这个不是绝对不行，而是从官方合约乘数和手续费看，性价比很差，这是基于交易规则做出的策略推论。 ([东京证券交易所][1])

#### B. 盘中回归子策略

适用时段：

* 10:30–14:30
* 夜盘中段波动趋稳时

逻辑：

1. 判断当天不是趋势日，而是震荡日。
2. 当价格偏离 VWAP 过大，但 NT 倍率没有同步扩张，说明更多是短线超买/超卖。
3. 在偏离 1.5~2 个短周期波动单位时反向进场。
4. 止盈回到 VWAP 附近，止损设在上一波极值之外。

这个子策略的作用不是追大行情，而是**补充趋势策略在震荡环境下的空窗期**。

### 2）确认引擎：TOPIX / miniTOPIX 趋势确认

如果你坚持看 TOPIX，我建议它更多作为**确认与过滤器**，而不是第一主战场；如果真的要单独做，优先用 **miniTOPIX**，因为 TOPIX 大合约的单笔波动太大。JPX 和 SBI 都有 miniTOPIX，合约单位是 TOPIX×1,000 日元，最小跳动 0.25 点。 ([东京证券交易所][3])

TOPIX 侧适合的逻辑是：

2. 同时 NT 倍率不大幅上升，说明不是少数日经权重股单独拉升，而是更广的市场在动。
3. 这种时候，TOPIX 多头胜率通常会比“追日经尖峰”更稳。

所以你的程序里可以这样写：

* **micro225 负责发出主信号**
* **TOPIX/miniTOPIX 负责做市场广度确认**
* 若 micro225 多头信号成立，但 TOPIX 明显偏弱，则降低仓位或放弃追价

### 3）相对价值引擎：NT 倍率回归

这是第二阶段再上的策略。

**思路**
做日经225和 TOPIX 的相对强弱，而不是单边赌方向。
SBI 官方就有 NT 倍率交易的说明：NT 上升时偏向“买日经、卖TOPIX”，NT 下降时偏向“卖日经、买TOPIX”。 ([sbisec.co.jp][4])

但你这里有一个现实问题：

**micro225 和 TOPIX 大合约不适合直接配对。**
因为 micro225 是指数×10 日元，而 TOPIX 大合约是指数×10,000 日元，量纲差太大。拿这两个直接做配对，腿数会很别扭，执行和风控都不好看。按合约规格推导，TOPIX 大合约名义规模会比 micro225 大很多倍，所以你会遇到“要么 TOPIX 太重，要么 micro 这边要堆很多手”的问题。这个是由合约乘数决定的。 ([东京证券交易所][1])

所以 NT 引擎更推荐：

* **日经225mini + miniTOPIX**
  或
* **日经225大合约 + TOPIX 大合约**

如果你一定要保留 micro225，那就把它定位成**方向交易腿**，不要拿它和 TOPIX 大合约硬做标准配对。

### 策略1: 趋势追踪型 (Trend Following)

**适用场景**: 趋势明确的市场环境

**核心逻辑**:
```
1. 入场条件:
   - 短周期均线上穿长周期均线 (金叉)
   - 动量指标(MACD/RSI)确认趋势方向
   - 成交量放大确认趋势有效性

2. 持仓管理:
   - 追踪止损(Trailing Stop)随趋势移动
   - 部分止盈在关键阻力位

3. 风控参数:
   - 单笔最大损失: 証拠金の10%以内
   - 最大同时持仓: 5枚(Micro225)
   - 日内最大亏损额: 账户2%
```

### 策略2: 区间震荡型 (Range Trading)

**适用场景**: 市场无明显趋势时

**核心逻辑**:
```
1. 入场条件:
   - 价格触及布林带上下轨
   - RSI超买(>70)/超卖(<30)
   - 价格回归均线时反向操作

2. 持仓管理:
   - 目标: 回归至中轨或对侧布林带
   - 固定止损在区间外侧

3. 风控参数:
   - 限値幅の80%以内设置止损
   - 临近交易结束时强制平仓
```

### 策略3: 跨期套利型 (Calendar Spread)

**适用场景**: 限月间价差出现偏离时

**核心逻辑**:
```
1. 入场条件:
   - 近月-远月价差超过历史均值±2σ
   - 价差向均值回归的概率高

2. 持仓管理:
   - 跨期对冲: 买近月+卖远月 (或反向)
   - 价差回归时同时平仓

3. 风控参数:
   - 最大价差敞口: 50万円相当
   - 临近限月前两周强制平仓
   - 注意SQ结算日效应
```

## 交易框架
建议把自动化逻辑拆成 5 层：**数据层、信号层、过滤层、执行层、风控层**。  
数据层负责实时行情、盘口、成交量、日内高低点、前收、夜盘状态；信号层负责判断趋势、均值回归、波动率扩张/收缩；过滤层负责避开开盘前后、重大事件、低流动时段；执行层负责下单、撤单、重挂、分批成交；风控层负责单笔止损、日内最大亏损、最大持仓、连续亏损熔断。

## micro225 策略逻辑
micro225 最适合做三类自动化逻辑：

1. **趋势突破**  
- 条件：价格突破前一交易日高点/低点，且成交量高于近 N 根均量。  
- 入场：突破后确认 1-2 根 K 线，再追单或回踩挂单。  
- 出场：ATR 止损、移动止盈、或者跌回突破位立即平仓。  
- 适合：早盘、夜盘开盘后、波动扩张阶段。

2. **均值回归**  
- 条件：价格偏离 VWAP、当日均线或布林带上轨/下轨，同时波动率没有继续扩张。  
- 入场：反向轻仓，分 2-3 次入场。  
- 出场：回到 VWAP、均线或中轴立即减仓。  
- 适合：震荡市、午盘、流动性稳定时段。

3. **事件驱动跟随**  
- 条件：日本宏观数据、美元/日元急变、海外股指夜盘联动。  
- 逻辑：用“方向确认 + 波动过滤”避免假突破。  
- 适合：重大数据前后，但必须缩仓。

micro225 的优势是**单张风险小**，可以用来验证参数、做分层仓位、做连续加减仓控制 。 [jpx.co](https://www.jpx.co.jp/english/derivatives/products/domestic/225micro-futures/01.html)

## TOPIX 策略逻辑
TOPIX 更偏向“市场整体风险偏好”和“广谱日股 Beta”：

1. **大盘趋势跟随**  
- 条件：指数站上/跌破日内 VWAP、20EMA、前高前低。  
- 配合：成交量、市场广度、银行/权重股同步性。  
- 特点：比 micro225 更稳定，但波动单位更大，仓位必须更小。

2. **对冲型策略**  
- 如果你有股票现货/主题仓，可以用 TOPIX 做 beta hedge。  
- 逻辑：当现货组合相关性较强时，用 TOPIX 对冲市场系统性风险。  
- 适合：组合管理，而不是纯方向交易。

3. **跨品种相对强弱**  
- 逻辑：micro225 与 TOPIX 同时监测，若 Nikkei 强于 TOPIX，偏向成长/权重主导；若 TOPIX 强于 Nikkei，偏向广谱修复。  
- 可以做成配对模型：同向时加仓，背离时减仓。

## SBI 执行逻辑
SBI 的先物・オプション API 可以直接接入外部工具做自动化，且 J-NET Cross 也可用于部分订单场景；SBI 的先物/期权相关页面还说明了逆指值、OCO、IFD、IFDOCO 等订单能力在服务中可用或已扩充 。 [sbisec.co](https://www.sbisec.co.jp/ETGate/?OutSide=on&getFlg=on&_ControlID=WPLETmgR001Control&_PageID=WPLETmgR001Mdtl20&_ActionID=DefaultAID&_DataStoreID=DSWPLETmgR001Control&burl=search_op&cat1=op&cat2=none&dir=service&file=op_service_05.html)

- 优先使用 **限价单**，避免滑点失控。
- 突破单可用 **stop 或 stop-limit 逻辑** 做触发后限价追单。
- 对于高波动时段，设置 **最大滑点阈值**，超出则放弃成交。

## 风控规则
建议至少加这些硬规则：

- 单笔最大亏损：账户净值的 0.2% 到 0.5%。
- 日内最大亏损：账户净值的 1% 到 2%，触发后停止交易。
- 单品种最大持仓：micro225 可多张，TOPIX 通常更少。
- 同时持有方向一致仓位时，合并计算总体风险。
- 遇到重大事件、流动性断层、涨跌停、系统异常，自动降级为只减仓不加仓。

### 🤖 核心策略逻辑：趋势跟踪

自动化交易的核心是让程序客观、一致地执行规则。这里推荐以 **趋势跟踪（Trend Following）** 作为基础逻辑，一个典型的策略逻辑可分解为以下四步：

1.  **判定市场趋势**：通过技术指标判断当前是上升、下降还是震荡趋势。
2.  **生成交易信号**：当价格回调至关键区域（如移动平均线）并符合趋势方向时，触发买入或卖出的初步信号。
3.  **信号过滤与确认**：引入辅助指标（如成交量、波动率等），过滤掉在震荡市中产生的无效信号，提高信号的可靠性。
4.  **执行交易与风险管理**：确认信号后执行开仓，并**同时设置止损（Stop Loss）和止盈（Take Profit）订单**。

### 📈 示例策略：基于移动平均线与波动率

以下示例策略可帮助更好地理解上述逻辑：

*   **趋势判定**：使用 **Volatility Quality Indicator (VQI)**。当 VQI > 0 时，判定为上升趋势；VQI < 0 时，判定为下降趋势。
*   **交易信号**：
    *   **买入（Long）**：在上升趋势（VQI > 0）中，当价格回调并触及某条移动平均线（如20周期SMA）的支撑位时，产生买入信号。
    *   **卖出（Short）**：在下降趋势（VQI < 0）中，当价格反弹并触及移动平均线的阻力位时，产生卖出信号。
*   **止损与止盈**：
    *   **固定止损**：设置一个固定的点数或亏损百分比作为止损位，例如，对于Micro225，可设置100点（约1,000日元亏损）。
    *   **时间止损**：如果持仓超过特定时间（如1小时）仍未盈利，则强制平仓离场。
    *   **跟踪止盈**：价格朝有利方向移动时，止盈点也随之移动以保护利润。例如，价格每上涨50点，止盈点就上移50点。

### 🧪 策略验证与风险评估

在投入实盘前，**策略验证是至关重要的一步**。

1.  **回测 (Backtesting)**：使用至少1-2年的历史数据，用Python（如`backtrader`库）等工具对策略进行回测。关注以下指标：
    *   **胜率 (Win Rate)**：盈利交易占总交易的比例。
    *   **盈亏比 (Risk-Reward Ratio)**：平均单笔盈利与平均单笔亏损的比值。
    *   **最大回撤 (Maximum Drawdown)**：账户资金从最高点到最低点的最大下跌幅度，这是衡量策略风险的关键指标。
    *   **夏普比率 (Sharpe Ratio)**：衡量每承受一单位风险所产生的超额回报。
2.  **模拟交易 (Paper Trading)**：回测结果满意后，通过API接入模拟账户进行至少1-3个月的实时模拟交易，验证策略在真实市场环境下的表现和API连接的稳定性。

### ⚙️ 自动化实现路径：SBI 先物・オプションAPI

SBI证券为个人开发者提供了先物・オプションAPI，可实现从行情获取到下单的完整自动化。

*   **API功能**：支持获取实时行情、账户信息查询、以及买卖、撤单等交易功能。
*   **开发选项**：
    1.  **完全自建**：使用 **Python** 调用API，灵活性最高，但对编程能力要求也高。
    2.  **Excel VBA**：SBI的“ネオトレAPI For Excel”允许通过Excel函数和VBA进行自动交易，上手相对简单，但功能受限。
    3.  **第三方工具**：SBI API也支持连接如“**オートレ225**”等第三方专业系统交易工具，可以省去开发过程，直接使用或购买现成的交易策略。

### 💡 重要风险提示

*   **市场风险**：先物交易自带杠杆，可能造成超出本金的损失。
*   **API延迟风险**：网络延迟可能导致信号抓取和下单不及时，造成滑点。
*   **流动性风险**：相比日经225mini等主流合约，Micro225和TOPIX先物（特别是非主力合约）的流动性可能稍差，导致无法以理想价格成交。
*   **系统风险**：程序漏洞、服务器宕机等都可能造成不可预料的损失。

因此，务必先用小额资金或模拟账户验证策略和系统，确认稳定可靠后再逐步投入。

### 二、 核心交易策略逻辑 (双轨制策略)

针对这两个品种的特性，可以部署两套互补的逻辑：

#### 策略 A：Micro 225 & TOPIX 跨品种统计套利 (Pairs Trading)
日经 225 偏向科技与出口导向型权重股，而 TOPIX 涵盖面更广，包含更多内需和传统行业。两者存在长期协整关系，但在极端行情或特定消息下会出现短期偏离。

* **因子构建**：
    * 实时计算两者的价格比率 (Price Ratio) 或构建线性回归模型计算价差 (Spread)。
    * 计算该价差的 $Z-Score$（过去 $N$ 分钟/小时的标准化偏离度）。
* **入场逻辑**：
    * 当 $Z-Score > 2.0$（即 Micro 225 相对高估，TOPIX 相对低估），系统自动发出指令：**做空 Micro 225，同时做多 TOPIX**。
    * 反之，当 $Z-Score < -2.0$ 时，反向开仓。
* **资金与仓位对冲**：
    * 需要根据两者的合约乘数和波动率进行 Beta 调整，确保两边腿（Legs）的资金敞口（Notional Value）相对中性。由于 Micro 225 面值极小，可以做到非常精确的对冲比例配平。
* **出场逻辑**：
    * 均值回归：当 $Z-Score$ 回落至 0 附近（如 0.5 到 -0.5 之间）时，双腿同时平仓获利。

#### 策略 B：早盘区间突破 (ORB) + VWAP 动量跟随
日系期指在早盘（东京时间 08:45 开盘后）波动率最大，非常适合捕捉日内趋势。

* **区间定义 (Opening Range)**：
    * 记录 08:45 至 09:00（或 09:15）的最高价 ($H$) 和最低价 ($L$)，形成初始震荡区间。
* **过滤条件 (VWAP 共振)**：
    * 实时计算当日成交量加权平均价 (VWAP)。这在判别大资金真实动向上非常关键。
* **入场逻辑**：
    * **做多**：当最新 Tick 向上突破 $H$，**且**当前价格位于 VWAP 之上（证明多头量能配合），市价或以买一价 ($Bid 1$) 追入多单。
    * **做空**：当最新 Tick 向下突破 $L$，**且**当前价格位于 VWAP 之下，开空单。
* **出场与止盈损**：
    * **动态追踪止损**：以前期低点或移动平均线作为追踪止损位。
    * **时间清仓**：日内策略严格在下午 15:10（日盘收盘前）执行强制平仓，不留隔夜敞口。

---

### 三、 微观结构与订单管理 (执行细节)

高频或自动化交易中，逻辑正确只是第一步，执行细节决定了最终的盈亏比：

1.  **Tick Size 差异处理**：
    * Micro 225 的最小变动价位是 5 日元，TOPIX 的最小变动价位是 0.5 点。在编写套利算法时，滑点预估和挂单价格必须严格按照各自的 Tick Size 进行 `round()` 处理，避免出现无效订单 (Invalid Price) 被 API 拒单。
2.  **挂单簿不平衡 (Order Book Imbalance, OBI)**：
    * 在 ORB 突破瞬间，读取盘口的买卖五档挂单量。如果突破方向的挂单墙极薄，而反方向积压大量限价单，可作为假突破的过滤信号，暂停开仓。
3.  **网格化建仓 (Grid / Scaling in)**：
    * 得益于 Micro 225 的小市值，在执行波段策略时，可以放弃一次性 All-in，采用金字塔式的网格分批建仓法，平滑入场成本。

---

### 四、 风控模块 (Kill Switch)

* **全局敞口限制**：严格限制同一方向的最大持仓手数。
* **连续亏损熔断**：单日内触发 3 次止损，或达到账户总权益的 2% 回撤，系统底层直接切断开仓 API 权限，只允许平仓。
* **断线重连与状态同步**：应对网络抖动，必须有一套状态机。如果在开仓套利双腿时，一腿成交、一腿飞单（Legging Risk），系统需在 500 毫秒内决定是市价追单还是平掉已成交的那一腿。

这套架构结合了统计套利的稳健和早盘突破的爆发力，并且在底层为可视化开发留出了空间。

### 2. **策略类实现**
```python
class FuturesTradingStrategy:
    def __init__(self, data, initial_balance=1000000):
        self.data = data
        self.initial_balance = initial_balance
        self.balance = initial_balance
        self.position = 0  # 0: 无仓位, 1: 多头, -1: 空头
        self.trades = []

    def calculate_indicators(self):
        # 计算移动平均线
        self.data['MA5'] = self.data['Close'].rolling(window=5).mean()
        self.data['MA20'] = self.data['Close'].rolling(window=20).mean()

        # 计算RSI
        delta = self.data['Close'].diff()
        gain = delta.where(delta > 0, 0)
        loss = -delta.where(delta < 0, 0)
        avg_gain = gain.rolling(window=14).mean()
        avg_loss = loss.rolling(window=14).mean()
        rs = avg_gain / avg_loss
        self.data['RSI'] = 100 - (100 / (1 + rs))

        # 计算ATR
        self.data['H-L'] = self.data['High'] - self.data['Low']
        self.data['H-PC'] = abs(self.data['High'] - self.data['Close'].shift())
        self.data['L-PC'] = abs(self.data['Low'] - self.data['Close'].shift())
        self.data['TR'] = self.data[['H-L', 'H-PC', 'L-PC']].max(axis=1)
        self.data['ATR'] = self.data['TR'].rolling(window=14).mean()

    def generate_signals(self):
        # 生成交易信号
        self.data['Signal'] = 0
        for i in range(1, len(self.data)):
            if self.data['MA5'].iloc[i-1] < self.data['MA20'].iloc[i-1] and self.data['MA5'].iloc[i] > self.data['MA20'].iloc[i] and self.data['RSI'].iloc[i] < 30:
                self.data['Signal'].iloc[i] = 1  # 买入信号
            elif self.data['MA5'].iloc[i-1] > self.data['MA20'].iloc[i-1] and self.data['MA5'].iloc[i] < self.data['MA20'].iloc[i] and self.data['RSI'].iloc[i] > 70:
                self.data['Signal'].iloc[i] = -1  # 卖出信号

    def execute_trades(self):
        # 执行交易
        for i in range(1, len(self.data)):
            if self.data['Signal'].iloc[i] == 1 and self.position <= 0:
                # 开多仓
                entry_price = self.data['Close'].iloc[i]
                self.position = 1
                self.trades.append({'type': 'buy', 'price': entry_price, 'time': self.data.index[i]})
            elif self.data['Signal'].iloc[i] == -1 and self.position >= 0:
                # 开空仓
                entry_price = self.data['Close'].iloc[i]
                self.position = -1
                self.trades.append({'type': 'sell', 'price': entry_price, 'time': self.data.index[i]})

            # 止损止盈逻辑
            if self.position == 1:
                if self.data['Close'].iloc[i] < entry_price - 2 * self.data['ATR'].iloc[i]:
                    # 止损
                    exit_price = entry_price - 2 * self.data['ATR'].iloc[i]
                    self.balance *= (exit_price / entry_price)
                    self.position = 0
                    self.trades.append({'type': 'sell', 'price': exit_price, 'time': self.data.index[i]})
                elif self.data['Close'].iloc[i] > entry_price + 3 * self.data['ATR'].iloc[i]:
                    # 止盈
                    exit_price = entry_price + 3 * self.data['ATR'].iloc[i]
                    self.balance *= (exit_price / entry_price)
                    self.position = 0
                    self.trades.append({'type': 'sell', 'price': exit_price, 'time': self.data.index[i]})
            elif self.position == -1:
                if self.data['Close'].iloc[i] > entry_price + 2 * self.data['ATR'].iloc[i]:
                    # 止损
                    exit_price = entry_price + 2 * self.data['ATR'].iloc[i]
                    self.balance *= (entry_price / exit_price)
                    self.position = 0
                    self.trades.append({'type': 'buy', 'price': exit_price, 'time': self.data.index[i]})
                elif self.data['Close'].iloc[i] < entry_price - 3 * self.data['ATR'].iloc[i]:
                    # 止盈
                    exit_price = entry_price - 3 * self.data['ATR'].iloc[i]
                    self.balance *= (entry_price / exit_price)
                    self.position = 0
                    self.trades.append({'type': 'buy', 'price': exit_price, 'time': self.data.index[i]})

    def backtest(self):
        self.calculate_indicators()
        self.generate_signals()
        self.execute_trades()
        final_balance = self.balance
        return final_balance, self.trades
```

### 核心策略：双市场统计套利 + 趋势跟随

鉴于你提到了两个品种：**TOPIX（大盘，代表日本整体经济，银行/旧经济权重高）** 和 **Nikkei 225 Micro（高科技/出口权重高，波动大）**。

**策略核心思想：** 利用两者的相关性进行对冲（赚价差），或者利用它们的背离进行单边投机（Micro 更适合做单边）。

#### 策略一：相关性配对交易
*   **适用场景：** 市场震荡，方向不明，但两个指数走势出现严重分歧。
*   **逻辑原理：** TOPIX 和 Nikkei 225 长期高度相关，短期会因板块轮动（如科技股涨、银行股跌）出现价差偏离。策略是“高卖低买”，等待价差回归。

**自动化交易逻辑流程：**

1.  **数据获取:**
    *   实时获取 `TOPIX Futures` 价格 ($P_t$) 和 `Nikkei 225 Micro Futures` 价格 ($P_n$)。
    *   建议使用 1分钟或 5分钟 K线。

2.  **价差计算与标准化:**
    *   由于点位不同（TOPIX 约 2000+，Nikkei 约 38000+），不能直接相减。需要计算 **Spread Ratio (价差比率)** 或 **Beta 调整后的价差**。
    *   公式：$Spread = P_n - (\beta \times P_t)$
        *   $\beta$ 可以通过过去 30 天的收盘价线性回归计算得出（通常 $\beta$ 约为 18-19 左右）。
    *   或者更简单：计算 $Z-Score (标准分数)$。
    *   $Z = \frac{CurrentSpread - MeanSpread}{StandardDeviation}$
    *   *MeanSpread* 和 *StdDev* 使用过去 20 根 K 线计算。

3.  **入场触发:**
    *   **做多 Micro / 做空 TOPIX (买入价差):**
        *   当 $Z < -2$ (Micro 相对 TOPIX 跌得过深，预期 Micro 会反弹得更快)。
        *   **动作 SBI:** 买入 1手 Nikkei 225 Micro，同时卖出约 18手 (或根据合约价值调整) TOPIX Mini/Futures。
    *   **做空 Micro / 做多 TOPIX (卖出价差):**
        *   当 $Z > +2$ (Micro 涨得过快，相对 TOPIX 超买)。
        *   **动作 SBI:** 卖出 1手 Nikkei 225 Micro，同时买入约 18手 TOPIX。

4.  **出场触发:**
    *   **止盈:** 当 $Z$ 回归到 0 附近（例如 $-0.5 < Z < 0.5$）。
        *   **动作:** 平掉所有仓位。
    *   **止损:** 当 $Z$ 进一步扩大（例如 $Z > 3$ 或 $Z < -3$），说明相关性失效，单边趋势爆发。
        *   **动作:** 立即止损平仓。

---

#### 策略二：东京开盘动量突破
*   **适用场景：** 美股隔夜波动较大，导致日经/东证大幅高开或低开。
*   **标的选择：** 主用 **Nikkei 225 Micro**（因为手续费低、保证金占用低，适合灵活试错）。
*   **逻辑原理：** 利用东京时间 9:00 开盘后的前 15-30 分钟确立当日方向。

**自动化交易逻辑流程：**

1.  **时间过滤:**
    *   仅在 **东京时间 08:55 - 09:05** 之间激活扫描。

2.  **隔夜情绪判断:**
    *   获取当前道琼斯指数和纳斯达克指数的涨跌幅 (%)。
    *   如果美股涨 > 1%，预判日股高开；跌 < -1%，预判日股低开。

3.  **形态确认:**
    *   计算 9:00 - 9:15 之间的 K 线实体。
    *   **做多逻辑:**
        *   条件 A: 美股隔夜上涨。
        *   条件 B: 9:00 开盘价 > 9:15 收盘价 (收盘站稳)。
        *   条件 C: 价格突破了昨日最高价 + 0.2% 的缓冲区。
        *   **动作:** 在 9:16 分，**市价单 (MO) 买入** 1手 Micro225。
    *   **做空逻辑:**
        *   条件 A: 美股隔夜下跌。
        *   条件 B: 9:00 开盘价 < 9:15 收盘价 (收阴)。
        *   条件 C: 价格跌破了昨日最低价 - 0.2%。
        *   **动作:** 在 9:16 分，**市价单 (MO) 卖出** 1手 Micro225。

4.  **动态出场:**
    *   **止损:** 入场价格的 +/- 30 个小点（Micro 约 300-500 日元波动）。
    *   **止盈 (移动止损):** 如果盈利超过 100 个点，设置止损线调整为保本；如果盈利超过 200 个点，设置跟踪止损为盈利 50 点。

### 针对 SBI 证券的执行技术架构

由于 SBI 证券的 API 限制，你不能像在 Interactive Brokers 那样直接运行 Python 脚本直接下单。你需要搭建一个中间层。

**系统架构建议：**

1.  **数据层:**
    *   不建议抓取 SBI 网页（太慢，容易被封）。
    *   建议使用 **Python + pandas** 连接免费的 WebSocket 数据源（如 Investing.com API 或 TradingView WebSocket）获取 TOPIX 和 Nikkei Micro 的实时价格。

2.  **策略引擎:**
    *   **语言:** Python 3.x。
    *   **库:** `pandas` (数据处理), `ta-lib` (技术指标), `numpy` (统计计算)。
    *   **运行:** 在本地电脑或云端服务器 (VPS) 运行脚本。

3.  **执行层:**
    *   **难点:** SBI 没有官方开放的 Python SDK 给散户。
    *   **解决方案:** 使用 **PyAutoGUI** 或 **Selenium** 模拟键盘/鼠标操作。这在 SBI 自动化交易中是常见的“偏方”。
    *   **流程:**
        1.  Python 计算出信号（例如“买入 Micro”）。
        2.  Python 调用 `pyautogui`，激活 SBI 的交易应用（如 SBI 证券的 HYBS 或专用交易终端）。
        3.  `pyautogui.click(坐标)` 点击代码输入框。
        4.  `pyautogui.write('Micro225')`。
        5.  `pyautogui.press('enter')` 确认下单。

**风险控制逻辑（必须在代码中硬编码）：**

*   **单日最大亏损:** 如果当日累计亏损 > 30,000 日元，程序自动停止交易，并强制关闭 SBI 应用。
*   **仓位限制:** Micro 单边持仓不得超过 3 手（防止黑天鹅事件爆仓）。
*   **API/网络故障保护:** 如果连续 1 分钟没有收到新价格数据，立即清仓离场。

### 总结与建议

1.  **Micro 225 vs TOPIX:**
    *   Micro 225 波动大，滑点相对高，适合单边趋势策略（策略二）。
    *   TOPIX 走势平稳，适合作为对冲品种（策略一）。
    *   **推荐:** 以**策略一（配对交易）**为主，因为它对冲了市场风险，胜率相对稳定，更适合自动化运行。

2.  **SBI 的局限性:**
    *   自动下单极其依赖屏幕坐标，一旦 SBI 升级软件界面或弹窗，你的自动交易程序可能会下错单。**务必先在模拟盘或最小手数进行长时间的实盘测试。**

3.  **代码逻辑片段示例 (Python - 策略一的核心):**

```python
import pandas as pd
import numpy as np

# 假设 df 是包含 TOPIX 和 Nikkei Micro 价格的 DataFrame
# 计算 Hedge Ratio (Beta)
model = np.polyfit(df['TOPIX'], df['NIKKEI_MICRO'], 1)
beta = model[0]

# 计算价差
df['Spread'] = df['NIKKEI_MICRO'] - (beta * df['TOPIX'])

# 计算 Z-Score
df['Mean'] = df['Spread'].rolling(window=20).mean()
df['Std'] = df['Spread'].rolling(window=20).std()
df['Z-Score'] = (df['Spread'] - df['Mean']) / df['Std']

# 信号生成
latest_z = df['Z-Score'].iloc[-1]

if latest_z < -2.0:
    signal = "BUY_MICRO_SELL_TOPIX"
    print(f"Signal: {signal}, Z-Score: {latest_z}")
    # 触发 PyAutoGUI 下单函数
elif latest_z > 2.0:
    signal = "SELL_MICRO_BUY_TOPIX"
    print(f"Signal: {signal}, Z-Score: {latest_z}")
    # 触发 PyAutoGUI 下单函数
else:
    print("No Action")
```

这套逻辑是最经典的量化入门思路，结合日本市场的特点进行了适配。请务必注意风控。下面是完整的策略梳理，分六个模块展开。

## 一、品种基础认识

**Micro日经225（マイクロ日経225先物）**
合约乘数 ¥100/点，最小变动 5 点（¥500/tick），保证金约 5–8 万日元，适合小资金。流动性集中在最近月合约，夜盘与 CME 日经期货联动紧密。

**TOPIX先物**
合约乘数 ¥10,000/点，最小变动 0.5 点（¥5,000/tick），保证金约 60–100 万日元。反映东证全市场，对国内基金持仓、ETF 申赎更敏感，波动率通常低于日经225。

两者相关性长期在 0.92 以上，但短期可出现价差偏移，提供配对交易机会。

---

## 二、核心策略模块

### 模块 A — 趋势跟踪（主策略）

适用品种：Micro225 为主，TOPIX 作方向确认

| 参数 | 设定 |
|---|---|
| 均线组合 | EMA 20 / 50 / 200（5m K线） |
| 趋势确认 | ADX(14) > 25 |
| 入场触发 | 价格突破 EMA50 + MACD 柱体方向一致 |
| 止损 | ATR(14) × 1.5，放在最近摆动低/高点之外 |
| 盈利目标 | +1R 减半仓，剩余用 trailing stop（EMA20 破位） |

**过滤规则**：若 TOPIX 方向与 Micro225 相反，暂停该笔交易。

---

### 模块 B — 动量反转（辅助策略）

适用场景：震荡区间、开盘后前 60 分钟的超卖/超买回归

入场条件（多头示例）：
1. RSI(14) < 30 在 15m 图
2. Stochastic(14,3,3) K 线从超卖区金叉
3. 价格位于 EMA200 上方（维持大趋势偏多）
4. 成交量当根 K 线 > 20 根均量的 1.2 倍

---

### 模块 C — 价差套利（N225 vs TOPIX）

计算公式：
```
spread_ratio = Micro225指数 / TOPIX指数
z_score = (spread_ratio - MA_20) / std_20
```

- z-score > +1.5σ → 做空 Micro225 / 做多 TOPIX
- z-score < -1.5σ → 做多 Micro225 / 做空 TOPIX
- z-score 回归 0 → 全平

对冲比例需根据各合约名义价值对齐（delta-neutral）。
